## Transport Ticketing & Payment System

Distributed backend platform integrating a **custom payment provider** with a ticketing system

Tech stack:
![PHP](https://img.shields.io/badge/PHP-777BB4?style=flat-square&logo=php&logoColor=white)
![MySQL](https://img.shields.io/badge/MySQL-4479A1?style=flat-square&logo=mysql&logoColor=white)
![Docker](https://img.shields.io/badge/Docker-2496ED?style=flat-square&logo=docker&logoColor=white)
![REST API](https://img.shields.io/badge/REST-API-black?style=flat-square)

Key architecture elements:

• Server-to-server REST integration  
• Bearer authentication for payment APIs  
• HMAC-SHA256 webhook verification  
• Idempotent webhook handling preventing duplicate payments  
• Atomic database operations ensuring financial consistency  
• Security mechanisms: CSRF protection, prepared SQL statements, server-side validation

Repo: https://github.com/Lorenzo-Benvenuti/Transport-Ticketing-and-Payment-System
